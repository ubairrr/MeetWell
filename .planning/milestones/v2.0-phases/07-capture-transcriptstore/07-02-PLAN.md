# Plan 07-02: DeepgramClient — single-channel WebSocket wrapper

## Objective

Create `DeepgramClient.ts` — a self-contained wrapper for one Deepgram Nova-3 WebSocket connection — and `SpeakerNormalizer.ts` — the per-channel speaker label mapping utility (D-09, D-10). These modules are used by `CaptureService` (07-05), one instance per channel. Both ship with unit tests that mock the Deepgram SDK so they run without a real API key.

## Context

**Deepgram SDK v5.4.0 exact API (verified from source):**

- Client: `new DeepgramClient({ apiKey })` imported from `@deepgram/sdk`
- Connection: `client.listen.v1.connect({ model: 'nova-3', diarize: 'true', mip_opt_out: true, encoding: 'linear16', sample_rate: 16000, interim_results: 'true', punctuate: 'true', Authorization: apiKey, reconnectAttempts: 0 })`
- Returns a `V1Socket` with `.on('open')`, `.on('message')`, `.on('error')`, `.on('close')`
- Audio send: `socket.sendMedia(buffer: Buffer | ArrayBuffer | ArrayBufferView)`
- `speech_final` detection: `data.type === 'Results' && data.speech_final === true` — this is the ONLY write trigger. `is_final` must not be used as a write trigger.
- Speaker label: `data.channel.alternatives[0].words[n].speaker` (0-indexed integer); only present in `speech_final` results
- Segment confidence: `data.channel.alternatives[0].confidence` (0.0–1.0)

**D-03/D-04 reconnect (CRITICAL):** Pass `reconnectAttempts: 0` to disable SDK built-in reconnect. Implement manually: max 3 attempts, 2-second fixed delay, then emit health `'error'`. The SDK default of 30 reconnect attempts conflicts with D-03.

**D-05:** No PCM buffering during reconnect — segments during gap are lost. `sendMedia` calls during reconnect window are silently dropped (socket is null).

**D-06:** Independent reconnect tracking per instance — each `DeepgramClient` manages its own counter.

**D-07 silence detection:** A 5-second timer that resets on every `sendMedia` call. When it fires, emit health `'silent'`. When audio resumes after silence, emit health `'healthy'`.

**`mip_opt_out: true` is hardcoded** — never read from config. This is a product-level commitment (DEC-02).

**SpeakerNormalizer (D-09/D-10):** Separate class with a `Map<number, string>` per instance. Mic channel: any speaker ID → `'You'`. System channel: speaker 0 → `'Speaker 1'`, speaker 1 → `'Speaker 2'`, etc. (N+1 mapping). `reset()` clears the map on reconnect.

## Tasks

### Task 1: SpeakerNormalizer utility

**File(s):** `src/main/capture/SpeakerNormalizer.ts`

**Action:** Create

**Details:**

Create `src/main/capture/` directory and `SpeakerNormalizer.ts`:

```typescript
export type ChannelType = 'mic' | 'system'

export class SpeakerNormalizer {
  private map: Map<number, string> = new Map()

  constructor(private readonly channel: ChannelType) {}

  normalize(speakerId: number | undefined): string {
    // Mic channel: always "You" regardless of speakerId (D-09)
    if (this.channel === 'mic') return 'You'

    // System channel: speakerId undefined defaults to 0
    const id = speakerId ?? 0
    if (!this.map.has(id)) {
      // Speaker 0 → "Speaker 1", speaker 1 → "Speaker 2", etc.
      this.map.set(id, `Speaker ${this.map.size + 1}`)
    }
    return this.map.get(id)!
  }

  reset(): void {
    // D-10: clear map on reconnect; speaker IDs restart from the new connection
    this.map.clear()
  }
}
```

Cap system speaker count at 8 (v1 spec): in `normalize`, if `this.map.size >= 8` and `id` is not in the map, map the new speaker to `'Speaker 8'` (reuse the cap slot) rather than creating a new entry.

### Task 2: DeepgramClient class

**File(s):** `src/main/capture/DeepgramClient.ts`

**Action:** Create

**Details:**

Exported types:

```typescript
export type HealthStatus = 'idle' | 'healthy' | 'silent' | 'error'

export interface SpeechFinalSegment {
  transcript: string
  speakerLabel: string   // normalized by SpeakerNormalizer
  channel: 'mic' | 'system'
  timestampStart: number
  timestampEnd: number
  confidence: number | null
}

export interface DeepgramClientOptions {
  apiKey: string
  channel: 'mic' | 'system'
  onSegment: (segment: SpeechFinalSegment) => void
  onHealthChange: (status: HealthStatus) => void
}
```

**`DeepgramClient` class:**

Constructor stores options, creates a `SpeakerNormalizer(channel)` instance. Private fields:

- `socket: V1Socket | null = null`
- `retryCount: number = 0`
- `readonly MAX_RETRIES = 3`
- `readonly RETRY_DELAY_MS = 2000`
- `silenceTimer: ReturnType<typeof setTimeout> | null = null`
- `readonly SILENCE_TIMEOUT_MS = 5000`

**`connect(meetingId: string): Promise<void>`** — opens a Deepgram connection. Implementation:

1. Import `DeepgramClient as DGClient` from `@deepgram/sdk`
2. `const client = new DGClient({ apiKey: this.options.apiKey })`
3. `this.socket = await client.listen.v1.connect({ model: 'nova-3', diarize: 'true', mip_opt_out: true, encoding: 'linear16', sample_rate: 16000, interim_results: 'true', punctuate: 'true', Authorization: this.options.apiKey, reconnectAttempts: 0 })`

Register handlers on `this.socket`:

- `'open'`: set `this.retryCount = 0`, call `this.options.onHealthChange('healthy')`, reset silence timer
- `'message'` handler: if `data.type === 'Results' && data.speech_final === true`: reset silence timer; extract `transcript`, `confidence`, dominant `speakerId` from `data.channel.alternatives[0]`; call `SpeakerNormalizer.normalize(speakerId)`; emit `onSegment` with a `SpeechFinalSegment`
- `'close'`: call `this.handleDisconnect()`
- `'error'`: call `this.handleDisconnect()`

**Speaker extraction from words:** The `speaker` field lives on each word. Use the speaker ID from the first word that has a `speaker` field: `const speakerId = data.channel.alternatives[0]?.words?.find(w => w.speaker !== undefined)?.speaker`. If no words have a speaker field, pass `undefined` to `normalize()`.

**Silence timer:** `resetSilenceTimer()` private method — clears existing timer, starts new `setTimeout` of `SILENCE_TIMEOUT_MS`. On timeout: `this.options.onHealthChange('silent')`. Call `resetSilenceTimer()` in the `'open'` handler and in the `'message'` handler (on every data event, even non-speech-final).

**`handleDisconnect(): Promise<void>`** — private async method:

```
if (this.retryCount >= this.MAX_RETRIES) {
  this.clearSilenceTimer()
  this.options.onHealthChange('error')   // D-04
  return
}
this.retryCount++
this.normalizer.reset()                  // D-10: reset speaker map on reconnect
await new Promise(r => setTimeout(r, this.RETRY_DELAY_MS))
await this.connect(/* meetingId stored in instance field */)
```

Store `meetingId` as an instance field on `connect()` call so `handleDisconnect` can re-call `connect`.

**`sendMedia(buffer: Buffer): void`** — if `this.socket` is null (during reconnect window, D-05): return silently. Otherwise: call `this.socket.sendMedia(buffer)`. Also call `resetSilenceTimer()` to track that audio is flowing.

**`disconnect(): Promise<void>`** — clear silence timer; if `socket` is non-null, call `this.socket.finish()` (or `this.socket.close()` — check SDK API; V1Socket should have a close/finish method). Set `this.socket = null`. Set `this.retryCount = MAX_RETRIES` to prevent handleDisconnect from reconnecting after explicit disconnect.

**`clearSilenceTimer()` private method** — `if (this.silenceTimer) { clearTimeout(this.silenceTimer); this.silenceTimer = null }`

**`resetSilenceTimer()` private method** — `this.clearSilenceTimer(); this.silenceTimer = setTimeout(() => this.options.onHealthChange('silent'), this.SILENCE_TIMEOUT_MS)`

**Details:**

Tests:
