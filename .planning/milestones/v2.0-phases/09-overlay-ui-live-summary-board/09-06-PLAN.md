# Plan 09-06: SettingsPanel + safeStorage + electron-store

## Goal

Implement the full settings panel — API key entry with safeStorage, overlay appearance sliders, the Gemini paid-plan warning, and the App.tsx gear icon that opens it from any state.

**On mount (useEffect with empty deps):**
1. Call `window.electronAPI.invoke('set-focusable', true)` to enable keyboard input.
2. Call `window.electronAPI.invoke('get-settings')` and populate state from the returned object.

**On unmount (useEffect cleanup):**
- Call `window.electronAPI.invoke('set-focusable', false)`.

Both in the same `useEffect`:
```typescript
useEffect(() => {
  window.electronAPI.invoke('set-focusable', true).catch(console.error)
  window.electronAPI.invoke('get-settings').then((settings: unknown) => {
    const s = settings as { overlayWidth: number; overlayOpacity: number; hasGeminiKey: boolean; hasDeepgramKey: boolean }
    setHasGeminiKey(s.hasGeminiKey)
    setHasDeepgramKey(s.hasDeepgramKey)
    setOverlayWidth(s.overlayWidth)
    setOverlayOpacity(s.overlayOpacity)
  }).catch(console.error)
  return () => {
    window.electronAPI.invoke('set-focusable', false).catch(console.error)
  }
}, [])
```

**Save key helper:**
```typescript
function saveKey(settingKey: 'gemini-api-key' | 'deepgram-api-key', value: string, setStatus: (s: 'idle' | 'saved' | 'error') => void) {
  if (!value.trim()) return
  window.electronAPI.invoke('set-setting', { key: settingKey, value: value.trim() })
    .then(() => {
      setStatus('saved')
      setTimeout(() => setStatus('idle'), 2000)
    })
    .catch(() => {
      setStatus('error')
      setTimeout(() => setStatus('idle'), 2000)
    })
}
```

**Save slider helper:**
```typescript
function saveSlider(settingKey: 'overlay-width' | 'overlay-opacity', value: number) {
  window.electronAPI.invoke('set-setting', { key: settingKey, value }).catch(console.error)
}
```

**Step 4 — Render SettingsPanel and gear button:**

In the App return, the overlay-root container is the positioning context. The SettingsPanel is rendered inside the container as an absolute overlay when `showSettings` is true.

For each render branch that returns a `<div id="overlay-root">`:
- Add `position: 'relative'` to the `overlayStyle` (or to the div directly) so that `SettingsPanel`'s absolute positioning is correctly contained
- Add `<GearButton onClick={() => setShowSettings(true)} />` as a child
- Add `{showSettings && <SettingsPanel onClose={() => setShowSettings(false)} />}` as the last child (so it layers over everything else)

Do this for all session states that have user-facing UI: Idle, PreCapture, Capturing (both CapturingScreen and board view), OnBreak, and Complete. Do not add it to the Processing state (transient spinner state — no meaningful settings action possible).

The cleanest implementation: define a `withOverlayChrome` wrapper function that takes the inner JSX and wraps it in the overlay-root div with the gear button and settings panel. Use this wrapper in each render branch instead of duplicating the gear + settings JSX.

Example wrapper:
```typescript
function withChrome(inner: JSX.Element): JSX.Element {
  return (
    <div id="overlay-root" style={{ ...overlayStyle, position: 'relative' }}>
      <GearButton onClick={() => setShowSettings(true)} />
      {inner}
      {showSettings && <SettingsPanel onClose={() => setShowSettings(false)} />}
    </div>
  )
}
```

Note: `withChrome` is a closure that captures `overlayStyle`, `showSettings`, `setShowSettings` from the enclosing `App()` scope. Define it inside `App()`.
