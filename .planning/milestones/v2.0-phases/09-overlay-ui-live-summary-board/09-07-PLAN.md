### T1: Fix all TypeScript compilation errors

**File:** All Phase 9 source files (modify as needed)

**Action:** Modify

**Details:**

Run `npm run typecheck` (or `tsc --noEmit`). Read the output. Fix every error reported.

Common issues to expect and resolve:

**Import path mismatches:** If `StoredSummaryCard` import paths are wrong (e.g., `'../../../shared/schemas/index'` when the actual project alias or relative depth is different), correct the path. Check how other components import from shared schemas.

**Missing or wrong type assertions:** If `window.electronAPI.on()` returns `void` but the hook treats it as a function, remove the cleanup call. If it returns an unsubscribe function, ensure the cleanup is typed correctly.

**LLMAdapter call signature mismatch:** If `SummaryCardTimer` calls `LLMAdapter.generate()` with wrong argument names, read `LLMAdapter.ts` and correct the call to match the actual signature.

**`Database.Database` import style:** If `better-sqlite3` types are imported differently in `SummaryCardStore.ts` vs the rest of the project, normalize to whichever style compiles correctly.

**`electron-store` types:** If electron-store v8 types are not found, install `@types/electron-store` or check whether v8 ships its own types. Run `npm install --save-dev @types/electron-store` if needed, but prefer the bundled types from the package itself.

Do not introduce type casts (`as unknown as X`) to silence errors â€” resolve the underlying type mismatch. Type casts are acceptable only for IPC payload types where the renderer receives `unknown` from the main process.

Iterate until `npm run typecheck` reports zero errors.
