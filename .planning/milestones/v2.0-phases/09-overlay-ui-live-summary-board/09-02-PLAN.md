# Plan 09-02: SummaryCardTimer + Main Process Wiring + Bug Fixes

## Goal

Create the SummaryCardTimer, wire it into the main process, implement the four stub IPC handlers (start-break, end-break, get-settings, set-setting), add the set-focusable channel, fix the OnBreak mouse-event bug, and set up API key loading at startup.

## Context

`src/main/index.ts` already has FSM wiring and stub handlers returning `undefined`. `src/preload/index.ts` already has all existing channels allowlisted. This plan creates the timer class, replaces every stub, and patches the two known bugs — making the main process fully functional for Phase 9. Plan 09-01 must be complete first (SummaryCardStore and SummaryCardSchema are imported here).

**Action:** Create

Implement class `SummaryCardTimer`:

**Imports:**
- `Database` from `'better-sqlite3'` (type-only if the project uses `import type`)
- `BrowserWindow` from `'electron'`
- `SummaryCardStore, StoredSummaryCard` from `'../store/SummaryCardStore'`
- `SummaryCard, SummaryCardSchema` from `'../../shared/schemas/index'`
- The LLMAdapter (check exact import path from `src/main/llm/LLMAdapter.ts`)

**Private fields:**
- `timeoutHandle: ReturnType<typeof setTimeout> | null = null`
- `currentMeetingId: string | null = null`
- `cardIndex: number = 0`
- `readonly INTERVAL_MS = 5 * 60 * 1000`

**Constructor:** `constructor(private readonly db: Database.Database, private readonly win: BrowserWindow, private readonly store: SummaryCardStore, private readonly llm: InstanceType<typeof LLMAdapter>)`

Note: check if LLMAdapter is exported as a class or a singleton. If it is a singleton, adjust accordingly.

**`start(meetingId: string): void`**
- Set `this.currentMeetingId = meetingId`
- Reset `this.cardIndex = 0`
- Call `this.scheduleNext()`

**`stop(): void`**
- If `this.timeoutHandle !== null`: call `clearTimeout(this.timeoutHandle)` and set `this.timeoutHandle = null`
- Set `this.currentMeetingId = null`

**`private scheduleNext(): void`**
- Set `this.timeoutHandle = setTimeout(async () => { ... }, this.INTERVAL_MS)`
- Inside the callback:
  - Wrap in try/catch — log errors with `console.error('[SummaryCardTimer] card generation failed:', err)`
  - In `finally`: if `this.timeoutHandle !== null` (meaning stop() was not called), call `this.scheduleNext()` again
- Call `await this.fire()` inside the try block

**`private async fire(): Promise<void>`**
- Guard: if `!process.env.GEMINI_API_KEY || !this.currentMeetingId` return early
- Compute `windowEndMs = Date.now()` and `windowStartMs = windowEndMs - this.INTERVAL_MS`
- Query transcript segments:
  ```sql
  SELECT * FROM transcript_segments
  WHERE meeting_id = ? AND created_at > ? AND created_at <= ?
  ORDER BY timestamp_start ASC
  ```
  Bind `[this.currentMeetingId, windowStartMs, windowEndMs]`. Use `this.db.prepare(...).all(...)`.
- If `segments.length === 0`: return (D-03 — no segments, skip LLM call, no card)
- Build `userContent`: join segments as one string with newline separators. Each segment formatted as:
  `[{row.speaker_label}] ({row.channel}): {row.text}`
- Call `this.llm.generate(...)` (or however the adapter is invoked — match the existing pattern exactly). Pass:
  - `schema`: `SummaryCardSchema`
  - `schemaName`: `'SummaryCardSchema'`
  - `systemPrompt`: the exact system prompt from the research notes (copy verbatim):
    ```
    You are a meeting assistant generating a brief summary of the last 5 minutes of a meeting transcript.
    Rules:
    1. Report only what was explicitly said. Do not infer, add context, or speculate.
    2. The topic_headline must be a short noun phrase (10 words or fewer) describing the main subject discussed.
    3. Each key_point must be a complete sentence directly reflecting something said in the transcript.
    4. If fewer than 3 distinct points were discussed, use all available points (minimum 1).
    5. speaker_contributions: include only speakers who actually spoke. One sentence per speaker.
    6. Output must match the JSON schema exactly. No additional fields.
    ```
  - `userContent`: the assembled segment string
- On success: call `this.store.saveCard(...)` with correct arguments. Compute:
  - `intervalStartSec = windowStartMs / 1000`
  - `intervalEndSec = windowEndMs / 1000`
  - `wallTimeLabel`: format `windowEndMs` as a local time string, e.g. `new Date(windowEndMs).toLocaleTimeString([], { hour: '2-digit', minute: '2-digit' })`
  - `cardIndex`: use `this.cardIndex`, then increment `this.cardIndex++`
- Push IPC: `this.win.webContents.send('summary-card-ready', storedCard)` where `storedCard` is the `StoredSummaryCard` returned by `getCardsForMeeting` after save, or reconstruct it inline. The simplest approach: after `saveCard` returns the id, call `this.store.getCardsForMeeting(this.currentMeetingId).find(c => c.id === id)` and send that. If null, log and skip send.

### T2: Wire SummaryCardTimer and implement stub handlers in main process

**File:** `src/main/index.ts`

**Action:** Modify

**Details:**

Read `src/main/index.ts` in full before making any change. Understand the current structure — where `createOverlayWindow` is defined, where IPC handlers are registered, where the FSM `onStateChange` callback is.

Also check `package.json` to confirm whether `electron-store` is present in `dependencies`. If it is not listed, add a note to the executor: run `npm install electron-store@8` before implementing, then proceed. Do not assume it is installed.

**Additions required (in order of initialization flow):**

1. Add imports at the top alongside existing imports:
   - `import Store from 'electron-store'` (default import)
   - `import { safeStorage } from 'electron'` (add to existing electron import destructure if present, or add new import)
   - `import { SummaryCardStore } from './store/SummaryCardStore'`
   - `import { SummaryCardTimer } from './context/SummaryCardTimer'`

2. After `app.whenReady()` resolves and before `createOverlayWindow()`, add:
   - `const electronStore = new Store()` — use `electronStore` (not `store`) to avoid collision with any existing variable named `store`
   - Define `function loadApiKeys(): void` that:
     - Checks `electronStore.has('gemini-api-key-encrypted')` — if true: reads the value as `Buffer`, calls `safeStorage.decryptString(value)`, sets `process.env.GEMINI_API_KEY = decrypted`
     - Checks `electronStore.has('deepgram-api-key-encrypted')` — if true: similar, sets `process.env.DEEPGRAM_API_KEY = decrypted`
     - Wrap each in try/catch and log errors with `console.error('[loadApiKeys]', err)` — a corrupt stored key must not crash startup
   - Call `loadApiKeys()` before `createOverlayWindow()`
   - Read overlay width: `const overlayWidth = electronStore.get('overlay-width', 380) as number`

3. Modify `createOverlayWindow` to accept `width: number = 380` parameter. Replace the hardcoded `OVERLAY_WIDTH` (or equivalent literal) with the `width` parameter in the `BrowserWindow` constructor options. Call site: `createOverlayWindow(overlayWidth)`.

4. After window creation, instantiate:
   - `const summaryCardStore = new SummaryCardStore(db)` — `db` is the existing initialized Database instance; find the variable name used in the current file
   - `const summaryCardTimer = new SummaryCardTimer(db, win, summaryCardStore, llmAdapter)` — `llmAdapter` is the existing LLMAdapter instance; find the variable name used in the current file. If LLMAdapter is a singleton module (not an instance), adjust accordingly.

5. In the FSM `onStateChange` callback:
   - When entering `'Capturing'` state: call `summaryCardTimer.start(currentMeetingId)` — find the variable name for the active meeting ID in the current file
   - When entering `'Processing'` state: call `summaryCardTimer.stop()`
   - When entering `'Idle'` state (session dismissed): call `summaryCardTimer.stop()` as a safety guard

6. Fix the OnBreak mouse-event bug:
   - Find the `setIgnoreMouseEvents` logic. It currently has a condition like `if (['Idle', 'PreCapture'].includes(newState))` or similar. The exact condition determines when click-through is active.
   - Add `'OnBreak'` to the list of INTERACTIVE states (states where mouse events are NOT ignored). This allows the "I'm Back" button to be clickable.
   - The logic reads: if the new state is one of the interactive states, call `win.setIgnoreMouseEvents(false)`; otherwise call `win.setIgnoreMouseEvents(true, { forward: true })`.

7. Replace `start-break` stub:
   - Transition FSM to `OnBreak` using the existing FSM transition method
   - Store `breakStartMs = Date.now()` as a module-level variable (declared alongside other session variables)
   - Return `undefined` or `{ ok: true }`

8. Replace `end-break` stub:
   - Transition FSM back to `Capturing` using the existing FSM transition method
   - Query `summaryCardStore.getCardsSince(currentMeetingId, breakStartMs)`
   - Send `win.webContents.send('break-assist-digest-ready', { cardsMissed: cards, isEmpty: cards.length === 0 })`
   - Reset `breakStartMs = 0`
   - Return `{ ok: true }`

9. Replace `get-settings` stub:
   - Return `{ overlayWidth: electronStore.get('overlay-width', 380), overlayOpacity: electronStore.get('overlay-opacity', 0.85), hasGeminiKey: electronStore.has('gemini-api-key-encrypted'), hasDeepgramKey: electronStore.has('deepgram-api-key-encrypted') }`

10. Replace `set-setting` stub:
    - Accept `payload: { key: string; value: unknown }`
    - Switch on `payload.key`:
      - `'gemini-api-key'`: validate `typeof payload.value === 'string'` and non-empty; call `safeStorage.encryptString(payload.value as string)` — returns a `Buffer`; store as `electronStore.set('gemini-api-key-encrypted', encrypted)`; set `process.env.GEMINI_API_KEY = payload.value as string`
      - `'deepgram-api-key'`: same pattern for `deepgram-api-key-encrypted` and `process.env.DEEPGRAM_API_KEY`
      - `'overlay-width'`: validate `typeof payload.value === 'number'`, clamp to 280–600; store as `electronStore.set('overlay-width', clamped)`
      - `'overlay-opacity'`: validate `typeof payload.value === 'number'`, clamp to 0.3–1.0; store as `electronStore.set('overlay-opacity', clamped)`
      - Default: return `{ error: 'unknown key' }`
    - Return `{ ok: true }` on success

11. Add `set-focusable` IPC handler:
    - `ipcMain.handle('set-focusable', (_event, payload: unknown) => { if (win) win.setFocusable(!!payload) })`
    - Place it alongside the other `ipcMain.handle` registrations
