    Create .planning/phases/03-deep-research/03-RSCH-02-REPORT.md with the following required sections:

    **## Decision Summary**: State the decision upfront — V1 diarization standard: Speaker labels without names (Speaker 1, Speaker 2, etc.), capped at 8 speakers reliably. Named attribution (Alice, Bob) is a v2 post-meeting confirmation flow.

    **## Deepgram Nova-3 Diarization Capabilities**: Document the confirmed capability data:
    - Supports up to 12 speakers (official limit; upgraded from 6–8 in Nova-2)
    - 53.1% overall accuracy improvement over prior version
    - 61.5% improvement specifically for meetings (most relevant domain)
    - Free with all ASR models — no additional per-minute charge
    - Trained on 100,000+ speakers across 80+ languages
    - Known weakness: overlapping speech — performance degrades when speakers talk simultaneously
    - No published per-speaker-count accuracy breakdown

    **## V1 Speaker Cap Recommendation**: State 8-speaker cap for v1 with rationale:
    - Deepgram officially supports 12, but no published accuracy breakdown by speaker count
    - Overlapping speech degradation is the primary quality cliff, not total count
    - 8 is a reasonable buffer below the 12 limit; covers the vast majority of real meeting scenarios
    - Standard team meeting: 4–6 people; max common meeting: 8; beyond 8 is conference/all-hands scenario

    **## V1 Diarization Flow**: Document the recommended flow as a diagram/step sequence:
    Audio capture (mic channel + system audio channel) → Deepgram Nova-3 streaming with diarization=true → Per-word speaker field ("speaker": 0, "speaker": 1, ...) → Transcript segments labeled "Speaker 1:", "Speaker 2:", ... → Mic-channel always = "You" / "Speaker 0"; System-audio channel = "Speaker 1", "Speaker 2" ... (diarized by Deepgram)

    **## V2 Named Attribution (Deferred)**: Document that named speaker attribution (Speaker 1 → "Alice") is a v2 differentiator. The v1 label foundation (Speaker 1/2/3) makes it straightforward to add without re-processing transcripts. Post-meeting confirmation flow: "Speaker 1 said 3 things — who was this?"

    **## Open Questions for PRD**: Note the open question from RESEARCH.md: does Deepgram diarization quality degrade notably between 4 and 8 speakers? No published per-speaker-count WER or DER data; 8 stands as conservative below the 12 limit pending empirical data.

    Cite data from 03-RESEARCH.md, noting [CITED] vs [ASSUMED] markers.
