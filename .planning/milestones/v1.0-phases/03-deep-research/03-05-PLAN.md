    **## Overview**: RSCH-06 uses two lenses: competitive research (what features do other tools support that our starter list misses?) and meeting-type discovery (what artifact/format differences do distinct meeting types require?). This is fully open discovery — no pre-specified use cases to validate.

    **## Meeting Type Taxonomy**: Include the 9-row table with columns: Meeting Type, Primary Artifact Need, Secondary Artifact, Key Differentiator. Types: Standup/Sync, 1:1 (Manager-Report), Design/Tech Review, Sales Call/Demo, Client/Stakeholder Briefing, All-hands/Town Hall, Interview, Training/Workshop, Retrospective.

    **## Competitive Feature Gap Analysis**: Include the complete feature gap table with columns: Feature, Competitive Standard, MeetingAssist v1 Status, Note. Cover all features from the research:
    - Table stakes (all tools): real-time transcription, meeting summary, action item extraction, speaker attribution
    - Table stakes (bot-based): search past meetings
    - Table stakes: calendar integration / .ics export
    - Differentiators: meeting-type-specific templates (v1 candidate), cross-meeting search/Q&A (v1 candidate via sqlite-vec)
    - MeetingAssist unique: live in-meeting assistant (yes — core), break assist (yes — core differentiator)
    - MeetingAssist unique: vision/screenshot analysis (yes — from DNA)
    - v2 only: CRM integration, Slack/Notion export, named speaker attribution, cross-platform, team collaboration, on-device/offline mode

    **## MeetingAssist's Unique Differentiators**: Highlight the two capabilities that competitors don't offer: (1) Live in-meeting assistant — real-time coaching, context chat, and break assist while the meeting is happening; (2) Break assist — functionality during meeting pauses. These are strategic differentiators, not just features.

    **## Notable Market Signals for PRD**: Five signals:
    1. Bot-free is a differentiator becoming table stakes — Fathom added silent capture April 2026; users strongly prefer no-bot where privacy or social dynamics matter
    2. Otter litigation (Aug 2025) and Fireflies BIPA lawsuit (Dec 2025) validate MeetingAssist's disclosed-consent-first posture (DEC-01) as both ethical and legally safer
    3. Granola's $43M raise (May 2025) validates significant investor confidence in macOS-native, bot-free approach — direct overlap with MeetingAssist's target
    4. Privacy is the top blocker — 46–50% of workers cite it as why they don't use these tools; MeetingAssist's encrypted local-first storage (DEC-02) directly addresses this
    5. Speaker attribution errors (11–13%) are a major quality pain point — trustworthy record value requires addressing this through diarization quality (RSCH-02)

    **## V2 Integrations (Not in V1 Scope)**: Per D-14, document these as v2 candidates surfaced for PRD awareness only: Slack (post summary to channel), Notion (create meeting page), HubSpot/Salesforce (CRM sync after sales calls), Google Calendar/Outlook (direct event creation beyond .ics), Zapier/Make (workflow automation).

    **## Recommendations for PRD-01 (MVP Boundary)**: Summarize key inputs for the PRD's feature spec:
    - Meeting-type templates: strong PRD-01 candidate based on competitive differentiation
    - Cross-meeting search/Q&A: strong candidate — sqlite-vec infrastructure designed in RSCH-05
    - All starred v2 features above should be tracked but explicitly excluded from v1 MVP boundary
