# Plan 09-03: LiveSummaryBoard + SummaryCard Components

## Goal

Create the two renderer components that visually display live summary cards in the overlay — `SummaryCard` for individual card rendering and `LiveSummaryBoard` for the scrollable stacked view.

## Context

These are pure display components. They receive already-typed `StoredSummaryCard` props and render them. No IPC calls, no hooks, no side effects beyond rendering. They will be wired into `App.tsx` in Plan 09-04. The overlay uses inline styles throughout (no CSS modules or Tailwind) — follow that convention exactly by reading any existing component before writing.

Read `src/renderer/src/components/CapturingScreen.tsx` and `src/renderer/src/components/ArtifactItem.tsx` to extract the exact inline style palette (background colors, text colors, font sizes, border radii, padding values) before writing a single line of JSX. Match the existing visual language exactly — do not introduce new color values.

## Tasks

### T1: Create SummaryCard component

**File:** `src/renderer/src/components/SummaryCard.tsx`

**Action:** Create

**Details:**

Import `StoredSummaryCard` from `'../../../shared/schemas/index'` (adjust the relative path to match the project's import convention — check an existing component that imports from shared to confirm the exact path).

Props interface:
```
interface SummaryCardProps {
  card: StoredSummaryCard
  isLatest?: boolean
}
```

Export default function `SummaryCard({ card, isLatest = false }: SummaryCardProps): JSX.Element`.

Layout (single `<div>` container, no external class names):

**Container styles** (inline object):
- `background`: `'rgba(0, 0, 0, 0.75)'` — match the existing overlay card background
- `borderRadius`: `'8px'`
- `padding`: `'10px 14px'`
- `marginBottom`: `'8px'`
- `borderLeft`: when `isLatest` is true, `'2px solid #2563eb'`; otherwise `'2px solid transparent'`
- `boxSizing`: `'border-box'` as `'border-box'`

**Timestamp label** (`<div>` at top):
- `fontSize`: `'10px'`
- `color`: use the muted secondary text color from the existing palette (e.g., `'rgba(255,255,255,0.45)'`)
- `marginBottom`: `'4px'`
- Content: `card.wall_time_label`

**Topic headline** (`<div>` below timestamp):
- `fontSize`: `'13px'`
- `fontWeight`: `600`
- `color`: white or the primary text color from the existing palette
- `marginBottom`: `'8px'`
- `lineHeight`: `'1.3'`
- Content: `card.topic_headline`

**Key points section**:
- Label `<div>`: `'Key points'` in small caps, muted color, `fontSize: '10px'`, `marginBottom: '4px'`, `textTransform: 'uppercase' as 'uppercase'`, `letterSpacing: '0.04em'`
- `<ul>` with `listStyle: 'disc'`, `paddingLeft: '16px'`, `margin: '0 0 8px 0'`
- Each `card.key_points` item renders as `<li>` with `fontSize: '12px'`, `lineHeight: '1.4'`, `marginBottom: '3px'`

**Speaker contributions section** (rendered only if `Object.keys(card.speaker_contributions).length > 0`):
- Label `<div>`: `'Speakers'` in same small-caps style as key points label
- For each entry in `card.speaker_contributions`: render `<div>` with `fontSize: '11px'`, `color`: muted text color, `marginBottom: '3px'`
- Format: `{speakerLabel}: {contribution}` — the speaker label bolded (`<span style={{ fontWeight: 600 }}>`) followed by the contribution text

Do not use any CSS class names. All styling via inline style objects. Do not import any CSS file.

### T2: Create LiveSummaryBoard component

**File:** `src/renderer/src/components/LiveSummaryBoard.tsx`

**Action:** Create

**Details:**

Import `StoredSummaryCard` from shared schemas (same path as T1). Import `SummaryCard` from `'./SummaryCard'`.

Props interface:
```
interface LiveSummaryBoardProps {
  cards: StoredSummaryCard[]  // caller provides newest-first order
}
```

Export default function `LiveSummaryBoard({ cards }: LiveSummaryBoardProps): JSX.Element`.

**Container div** (fills the space given by App.tsx parent):
- `display: 'flex'`, `flexDirection: 'column'`, `height: '100%'`, `overflow: 'hidden'`

**Header row** (inside the container, before the scrollable list):
- `display: 'flex'`, `alignItems: 'center'`, `justifyContent: 'space-between'`
- `padding: '6px 12px'`
- `borderBottom: '1px solid rgba(255,255,255,0.08)'`
- `flexShrink: 0` — must not shrink when card list is long
- Left: `<span>` text `'Live Summary'` in small-caps style, `fontSize: '11px'`, `letterSpacing: '0.05em'`, `color: 'rgba(255,255,255,0.5)'`, `textTransform: 'uppercase' as 'uppercase'`
- Right: card count badge `<span>` — background `'rgba(37,99,235,0.25)'`, `color: '#60a5fa'`, `fontSize: '10px'`, `padding: '1px 6px'`, `borderRadius: '10px'` — content: `cards.length === 1 ? '1 card' : \`${cards.length} cards\``

**Scrollable card list** (below header):
- `flex: 1`, `overflowY: 'auto' as 'auto'`, `padding: '8px 10px'`
- Map over `cards`: render `<SummaryCard key={card.id} card={card} isLatest={index === 0} />`
- The first card in the array (index 0) is newest, so `isLatest={true}` for index 0 only

No empty-state rendering — when `cards` is empty the board is never shown (App.tsx guards this). Trust the caller.

Do not add scroll-restoration logic or virtualization. The overlay is narrow and card count is bounded by meeting duration — plain DOM scroll is sufficient.
