# Plan 07-04: AudioWorklet mic bridge (renderer-side)

## Objective

Create the renderer-side mic capture pipeline: an `AudioWorkletProcessor` that converts Float32 PCM to Int16 in 250ms batches (D-01, D-02) and a `MicCapture` module that manages the `AudioContext`, worklet lifecycle, and forwards each batch to the main process via `window.electronAPI.invoke('mic-audio-chunk', arrayBuffer)`. The worklet file is plain JavaScript (AudioWorklet context restrictions prohibit ES modules).

## Context

**D-01:** Float32→Int16 conversion happens in the worklet processor, not the main process. The main process receives ready-to-stream `Int16` bytes.

**D-02 (research-corrected):** Use `AudioContext({ sampleRate: 16000 })`. At 16kHz, 250ms = 4,000 samples per flush. Each `AudioWorklet` quantum is 128 frames at 16kHz = 8ms. ~31 quanta per flush (4,000 / 128 = 31.25). The CONTEXT.md "~35 quanta" is consistent with 16kHz and supersedes the "44.1kHz / 11,025 samples" description in the raw decision text — research resolved this as a 16kHz assumption.

**IPC binary transfer:** Send `int16Array.buffer` (the underlying `ArrayBuffer`), NOT a `Uint8Array`. Electron bug #35152 causes `Uint8Array` to arrive as a plain `Object` in the main process. `ArrayBuffer` passes through Structured Clone correctly.

**AudioWorklet module URL in Electron + Vite:** The worklet file must be:
1. A plain `.js` file (not TypeScript — worklet scope lacks standard globals)
2. Placed in `src/renderer/public/` so Vite serves it at `/audio/mic-processor.worklet.js` during dev
3. Available at the same URL path in production (Vite copies `public/` to the renderer output)

Create the worklet file at `src/renderer/public/audio/mic-processor.worklet.js` and load it via `audioContext.audioWorklet.addModule('/audio/mic-processor.worklet.js')`. This path works in both dev (Vite dev server) and production (file served from renderer bundle root).

**CAPT-09:** The worklet processor never stores Float32 samples beyond the current accumulation buffer. Once the `Int16Array` is posted with `transfer: [int16.buffer]`, ownership transfers to the main thread — the worklet buffer is released.

**MicCapture module** (`src/renderer/src/audio/MicCapture.ts`): imperative start/stop API, not a React hook. `CapturingScreen` (07-06) calls `startMicCapture()` on mount and `stopMicCapture()` on unmount. The module is not a singleton — it returns a `MicCaptureHandle` with a `stop()` method.

## Tasks

### Task 1: AudioWorkletProcessor (plain JS)

**File(s):** `src/renderer/public/audio/mic-processor.worklet.js`

**Action:** Create

**Details:**

Create directory `src/renderer/public/audio/` and file `mic-processor.worklet.js`:

```javascript
// AudioWorkletProcessor: Float32 → Int16 PCM conversion
// Runs in the AudioWorklet global scope — no ES modules, no import/export
// D-01: conversion happens here, before postMessage
// D-02: flush every 4,000 samples (250ms at 16kHz)

const TARGET_SAMPLES = 4000  // 250ms at sampleRate:16000

class MicProcessor extends AudioWorkletProcessor {
  constructor() {
    super()
    this._buffer = new Float32Array(TARGET_SAMPLES)
    this._offset = 0
  }

  process(inputs) {
    const channel = inputs[0]?.[0]
    if (!channel || channel.length === 0) return true

    let i = 0
    while (i < channel.length) {
      const remaining = TARGET_SAMPLES - this._offset
      const toCopy = Math.min(remaining, channel.length - i)

      this._buffer.set(channel.subarray(i, i + toCopy), this._offset)
      this._offset += toCopy
      i += toCopy

      if (this._offset >= TARGET_SAMPLES) {
        // Convert Float32 → Int16 (D-01)
        const int16 = new Int16Array(TARGET_SAMPLES)
        for (let j = 0; j < TARGET_SAMPLES; j++) {
          const s = Math.max(-1, Math.min(1, this._buffer[j]))
          int16[j] = s < 0 ? (s * 32768) : (s * 32767)
        }

        // Transfer ownership to main thread — zero-copy, CAPT-09 compliant
        this.port.postMessage({ type: 'audio', buffer: int16.buffer }, [int16.buffer])

        // Reset accumulation buffer
        this._buffer = new Float32Array(TARGET_SAMPLES)
        this._offset = 0
      }
    }

    return true  // Keep processor alive
  }
}

registerProcessor('mic-processor', MicProcessor)
```

No imports, no exports. The file must be valid ES5/ES6 without module syntax.

### Task 2: MicCapture module

**File(s):** `src/renderer/src/audio/MicCapture.ts`

**Action:** Create

**Details:**

Create directory `src/renderer/src/audio/` and `MicCapture.ts`:

```typescript
export interface MicCaptureHandle {
  stop(): Promise<void>
}

export async function startMicCapture(): Promise<MicCaptureHandle> {
  // Request mic permission
  const stream = await navigator.mediaDevices.getUserMedia({
    audio: {
      echoCancellation: false,  // Do not alter the raw audio
      noiseSuppression: false,
      autoGainControl: false,
    },
    video: false,
  })

  // AudioContext at 16kHz — no downsampling needed (D-02 research resolution)
  const ctx = new AudioContext({ sampleRate: 16000 })

  // Load worklet — path must be absolute from renderer root
  await ctx.audioWorklet.addModule('/audio/mic-processor.worklet.js')

  const source = ctx.createMediaStreamSource(stream)
  const workletNode = new AudioWorkletNode(ctx, 'mic-processor')

  workletNode.port.onmessage = (event: MessageEvent) => {
    if (event.data?.type !== 'audio') return
    // event.data.buffer is a transferred ArrayBuffer (not Uint8Array)
    // IMPORTANT: send ArrayBuffer, not Uint8Array — Electron IPC bug #35152
    window.electronAPI.invoke('mic-audio-chunk', event.data.buffer).catch((err: unknown) => {
      console.error('[MicCapture] IPC error:', err)
    })
  }

  source.connect(workletNode)
  // Do NOT connect workletNode to ctx.destination — we capture only, no playback

  return {
    async stop(): Promise<void> {
      workletNode.port.onmessage = null
      workletNode.disconnect()
      source.disconnect()
      stream.getTracks().forEach(t => t.stop())
      await ctx.close()
    },
  }
}
```

The `electronAPI` type is declared in `src/renderer/env.d.ts` (Phase 6). If it lacks `invoke` typing for `'mic-audio-chunk'`, the call still works at runtime — the contextBridge allowlist is the enforcement layer.

### Task 3: Unit tests for the Float32→Int16 conversion logic

**File(s):** `tests/unit/mic-processor.test.ts`

**Action:** Create

**Details:**

The worklet processor class runs in a restricted AudioWorklet scope and cannot be imported as a Node module. Test the Float32→Int16 conversion logic in isolation.

Extract the conversion logic into a testable pure function inline in the test, matching the exact algorithm in the worklet:

```typescript
function float32ToInt16(samples: Float32Array): Int16Array {
  const int16 = new Int16Array(samples.length)
  for (let i = 0; i < samples.length; i++) {
    const s = Math.max(-1, Math.min(1, samples[i]))
    int16[i] = s < 0 ? (s * 32768) : (s * 32767)
  }
  return int16
}
```

Tests:

1. `float32 0.0 → int16 0` — `float32ToInt16(new Float32Array([0.0]))[0] === 0`
2. `float32 1.0 → int16 32767` — `float32ToInt16(new Float32Array([1.0]))[0] === 32767`
3. `float32 -1.0 → int16 -32768` — `float32ToInt16(new Float32Array([-1.0]))[0] === -32768`
4. `clipping above 1.0 is clamped to 32767` — `float32ToInt16(new Float32Array([1.5]))[0] === 32767`
5. `clipping below -1.0 is clamped to -32768` — `float32ToInt16(new Float32Array([-1.5]))[0] === -32768`
6. `chunk size: 4000 samples at 16kHz = 8000 bytes` — create `Int16Array(4000)`; `int16.buffer.byteLength === 8000`
7. `full silence frame (all zeros) → all int16 zeros` — `Float32Array(4000)` (zeroed); all int16 values are 0

Use Vitest `describe/it/expect`. No DOM or Electron APIs needed — pure math tests.
