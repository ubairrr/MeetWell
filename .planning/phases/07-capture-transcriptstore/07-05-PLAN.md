# Plan 07-05: CaptureService — orchestrate both channels

## Objective

Create `CaptureService.ts` — the main-process orchestrator that starts `SystemAudioSource` + two `DeepgramClient` instances (mic and system), handles `speech_final` segments end-to-end (normalize speaker → insert to DB → push IPC to renderer), tracks channel health, and implements the CAPT-03 Chromium fallback path when `audiotee` fails.

## Context

`CaptureService` is the sole owner of both audio capture pipelines. It is instantiated once in `src/main/index.ts` (07-07) and called only from `SessionManager` state-transition hooks — never directly from IPC handlers.

**Start sequence (`startCapture(meetingId)`):**

1. Call `transcriptStore.createMeeting(meetingId, Date.now())`
2. Create `DeepgramClient` instances for both channels (mic and system), passing `onSegment` and `onHealthChange` callbacks
3. `await deepgramMic.connect(meetingId)` and `await deepgramSystem.connect(meetingId)` — these are concurrent (use `Promise.all`)
4. Start `SystemAudioSource` — `await systemAudio.start()`
5. Wire `SystemAudioSource 'data'` event: `deepgramSystem.sendMedia(buffer)`
6. Wire `SystemAudioSource 'fallback-needed'` event: activate `ChromiumFallback` path and push `capture-health-update` with `{ channel: 'system', status: 'error' }` indicating fallback active
7. Set health state for both channels to `'idle'` initially; update to `'healthy'` when Deepgram `open` fires

**Stop sequence (`stopCapture()`):**

1. `await systemAudio.stop()`
2. `await deepgramSystem.disconnect()` and `await deepgramMic.disconnect()` — concurrent
3. Clear silence timers (handled inside `DeepgramClient.disconnect()`)
4. Return `meetingId` (so index.ts can pass it to the FSM or next phase)

**`handleMicChunk(buffer: ArrayBuffer): void`** — called from the IPC handler in `index.ts` (07-07). Forwards to `deepgramMic.sendMedia(Buffer.from(buffer))`. If mic channel is not connected yet, drop silently.

**`onSegment` callback (shared by both channels):** receives a `SpeechFinalSegment` from `DeepgramClient`. Constructs a `TranscriptSegmentRow`, calls `transcriptStore.appendSegment(row)`, then calls `win.webContents.send('transcript-segment', { id, text, speakerLabel, channel, timestampStart, timestampEnd, confidence })`.

**`onHealthChange` callback:** receives `(status: HealthStatus)` and channel identity from the closure. Calls `win.webContents.send('capture-health-update', { channel: 'mic'|'system', status })`.

**CAPT-03 Chromium loopback fallback (minimal implementation):** If `SystemAudioSource` emits `'fallback-needed'`, `CaptureService` pushes a `capture-health-update` with `{ channel: 'system', status: 'error' }` and logs the fallback condition. The actual Chromium `getUserMedia` fallback path is in the renderer and triggered via a `start-system-audio-fallback` IPC event (not currently in the preload allowlist — add a note that this is a v1 limitation: the fallback signals an error state rather than activating the Chromium path automatically, since that requires renderer action and a preload channel not yet wired). The health dot turns red; the user can continue with mic-only.

**CAPT-09:** `CaptureService` never stores PCM buffers. The `Buffer` from `SystemAudioSource 'data'` is passed directly to `deepgramSystem.sendMedia(buffer)` and then the reference is released.

**Segment ID generation:** Use `crypto.randomUUID()` (available in Node 24 — no `uuid` package needed).

**`startCapture(meetingId: string): Promise<void>`:**

```typescript
async startCapture(meetingId: string): Promise<void> {
  this.currentMeetingId = meetingId
  this.transcriptStore.createMeeting(meetingId, Date.now())

  this.deepgramMic = new DeepgramClient({
    apiKey: this.apiKey,
    channel: 'mic',
    onSegment: (seg) => this.handleSegment(seg),
    onHealthChange: (status) => this.pushHealthUpdate('mic', status),
  })

  this.deepgramSystem = new DeepgramClient({
    apiKey: this.apiKey,
    channel: 'system',
    onSegment: (seg) => this.handleSegment(seg),
    onHealthChange: (status) => this.pushHealthUpdate('system', status),
  })

  await Promise.all([
    this.deepgramMic.connect(meetingId),
    this.deepgramSystem.connect(meetingId),
  ])

  this.systemAudio = new SystemAudioSource()

  this.systemAudio.on('data', (buffer: Buffer) => {
    // CAPT-09: forward immediately, do not store
    this.deepgramSystem?.sendMedia(buffer)
  })

  this.systemAudio.on('fallback-needed', (err: Error) => {
    console.warn('[CaptureService] audiotee unavailable, fallback needed:', err.message)
    // Push error status — Chromium loopback fallback requires renderer action (v1 limitation)
    this.pushHealthUpdate('system', 'error')
  })

  this.systemAudio.on('error', (err: Error) => {
    console.error('[CaptureService] SystemAudioSource error after retries:', err.message)
    this.pushHealthUpdate('system', 'error')
  })

  await this.systemAudio.start()
}
```

**`stopCapture(): Promise<string | null>`:**

```typescript
async stopCapture(): Promise<string | null> {
  const meetingId = this.currentMeetingId

  await Promise.all([
    this.systemAudio?.stop(),
    this.deepgramSystem?.disconnect(),
    this.deepgramMic?.disconnect(),
  ])

  this.systemAudio = null
  this.deepgramMic = null
  this.deepgramSystem = null
  this.currentMeetingId = null

  return meetingId
}
```

**`handleMicChunk(buffer: ArrayBuffer): void`:**

```typescript
handleMicChunk(buffer: ArrayBuffer): void {
  if (!this.deepgramMic) return
  this.deepgramMic.sendMedia(Buffer.from(buffer))
}
```

**`handleSegment(seg: SpeechFinalSegment): void` (private):**

```typescript
private handleSegment(seg: SpeechFinalSegment): void {
  if (!this.currentMeetingId) return

  const row: TranscriptSegmentRow = {
    id: crypto.randomUUID(),
    meetingId: this.currentMeetingId,
    speakerLabel: seg.speakerLabel,
    channel: seg.channel,
    timestampStart: seg.timestampStart,
    timestampEnd: seg.timestampEnd,
    text: seg.transcript,
    isSpeechFinal: 1,
    confidence: seg.confidence,
    createdAt: Date.now(),
  }

  this.transcriptStore.appendSegment(row)

  // Push to renderer — display-only, no raw audio (CAPT-09)
  this.win.webContents.send('transcript-segment', {
    id: row.id,
    text: row.text,
    speakerLabel: row.speakerLabel,
    channel: row.channel,
    timestampStart: row.timestampStart,
    timestampEnd: row.timestampEnd,
    confidence: row.confidence,
  })
}
```

**`pushHealthUpdate(channel: 'mic' | 'system', status: HealthStatus): void` (private):**

```typescript
private pushHealthUpdate(channel: 'mic' | 'system', status: HealthStatus): void {
  if (!this.win.isDestroyed()) {
    this.win.webContents.send('capture-health-update', { channel, status })
  }
}
```

### Task 2: Unit tests for CaptureService

**File(s):** `tests/unit/CaptureService.test.ts`

**Action:** Create

**Details:**

Mock `DeepgramClient`, `SystemAudioSource`, `TranscriptStore`, and `BrowserWindow` using `vi.mock` / `vi.fn()`.

Tests:

1. `startCapture creates a meeting row` — call `captureService.startCapture('mtg-1')`; verify `transcriptStore.createMeeting` was called with `'mtg-1'` and a number timestamp.

2. `startCapture connects both Deepgram channels concurrently` — verify both `deepgramMic.connect('mtg-1')` and `deepgramSystem.connect('mtg-1')` were called.

3. `handleMicChunk forwards buffer to mic DeepgramClient` — call `startCapture` then `handleMicChunk(new ArrayBuffer(8000))`; verify `deepgramMic.sendMedia` called with a `Buffer`.

4. `handleMicChunk before startCapture is a no-op` — call `handleMicChunk` without starting; no error thrown.

5. `speech_final segment from mic channel inserts to DB and sends IPC` — trigger `onSegment` callback (captured from `DeepgramClient` constructor options) with a `SpeechFinalSegment`; verify `transcriptStore.appendSegment` called and `win.webContents.send('transcript-segment', ...)` called.

6. `health update from system channel pushes capture-health-update IPC` — trigger `onHealthChange('silent')` for system channel; verify `win.webContents.send('capture-health-update', { channel: 'system', status: 'silent' })` called.

7. `D-06: mic channel failure does not affect system channel` — trigger system `DeepgramClient` `onHealthChange('error')`; verify mic `DeepgramClient.disconnect()` was NOT called.

8. `stopCapture disconnects both channels and systemAudio` — call `startCapture` then `stopCapture`; verify `systemAudio.stop()`, `deepgramSystem.disconnect()`, `deepgramMic.disconnect()` all called.
