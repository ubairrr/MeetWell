# Plan 07-01: DB Migration + TranscriptStore

## Objective

Add the `confidence REAL` column to `transcript_segments` via an idempotent migration and implement `TranscriptStore` — the single write path between Deepgram `speech_final` events and the encrypted SQLCipher database. All downstream plans that produce transcript rows depend on this module existing first.

## Context

The `transcript_segments` table was created in Phase 6 via `CREATE TABLE IF NOT EXISTS`, so the DDL will not re-execute. CAPT-05 requires a confidence score per segment (from `result.channel.alternatives[0].confidence`). The migration must be idempotent: check `PRAGMA table_info(transcript_segments)` for a column named `confidence` and only run `ALTER TABLE` if it is absent.

`TranscriptStore` wraps `better-sqlite3-multiple-ciphers` synchronous prepared statements — no async needed. The module accepts the `Database.Database` instance returned by `openDatabase()` in `src/main/store/db.ts` (already called in `src/main/index.ts`).

`TranscriptSegmentRow` is the authoritative TypeScript type for a DB row. Downstream plans (07-05, 07-07) import this type and the `TranscriptStore` class.

The `meetings` table requires a row before any `transcript_segments` row can be inserted (foreign key). `TranscriptStore.createMeeting()` handles this so `CaptureService` (07-05) has one call site to create the meeting record.

## Tasks

### Task 1: DB Migration — add `confidence REAL` column

**File(s):** `src/main/store/db.ts`

**Action:** Modify

**Details:**

Add a `runMigrations(db: Database.Database): void` function exported from `db.ts`. Call it at the end of `openDatabase()`, after `db.exec(ALL_DDLS)`.

`runMigrations` must be idempotent. Implementation:

1. Query `PRAGMA table_info(transcript_segments)` — returns one row per column with a `name` field.
2. Check whether any row has `name === 'confidence'`.
3. If absent, execute: `ALTER TABLE transcript_segments ADD COLUMN confidence REAL`
4. Wrap in try/catch — swallow `duplicate column name` errors silently (safe re-run guard).

Call site in `openDatabase()` — append after `db.exec(ALL_DDLS)`:

```
runMigrations(db)
```

Also update `ALL_DDLS` to include `confidence REAL` in the `transcript_segments` DDL for fresh installs (new databases will have the column from the start):

Add `confidence REAL` as a column in the `CREATE TABLE IF NOT EXISTS transcript_segments` block, positioned after `is_speech_final INTEGER NOT NULL DEFAULT 1`.

The migration guards against the `CREATE TABLE IF NOT EXISTS` case (existing DB) while `ALL_DDLS` handles fresh installs.

### Task 2: TranscriptStore module + TypeScript interface

**File(s):** `src/main/transcript/TranscriptStore.ts`

**Action:** Create

**Details:**

Create `src/main/transcript/` directory and `TranscriptStore.ts` with the following:

**`TranscriptSegmentRow` interface** (exported):

```typescript
export interface TranscriptSegmentRow {
  id: string
  meetingId: string
  speakerLabel: string
  channel: 'mic' | 'system'
  timestampStart: number
  timestampEnd: number
  text: string
  isSpeechFinal: 1
  confidence: number | null
  createdAt: number
}
```

**`TranscriptStore` class** (exported):

Constructor: `constructor(private db: Database.Database)`

Methods:

- `createMeeting(meetingId: string, startedAt: number): void` — inserts a row into `meetings` with `id`, `started_at`, and `created_at`. Uses a prepared statement. `title` is null at creation time (updated later by Phase 8).

- `appendSegment(row: TranscriptSegmentRow): void` — inserts a row into `transcript_segments`. Maps TypeScript camelCase fields to snake_case DB columns: `meetingId → meeting_id`, `speakerLabel → speaker_label`, `timestampStart → timestamp_start`, `timestampEnd → timestamp_end`, `isSpeechFinal → is_speech_final`. Uses a parameterized prepared statement — never string-interpolates `row.text` into SQL.

- `getSegmentsByMeeting(meetingId: string): TranscriptSegmentRow[]` — `SELECT ... FROM transcript_segments WHERE meeting_id = ? ORDER BY timestamp_start ASC`. Returns array of `TranscriptSegmentRow`. Maps snake_case columns back to camelCase.

Import `Database` type from `better-sqlite3-multiple-ciphers` for the constructor parameter type annotation. Use `import type Database from 'better-sqlite3-multiple-ciphers'`.

Prepare statements in the constructor (not on every call): `this.insertMeetingStmt = db.prepare(...)`, `this.insertSegmentStmt = db.prepare(...)`, `this.getSegmentsStmt = db.prepare(...)`.

### Task 3: Unit tests for TranscriptStore

**File(s):** `tests/unit/TranscriptStore.test.ts`

**Action:** Create

**Details:**

Use Vitest. Create an in-memory SQLite database for tests (no encryption needed in unit tests): `new Database(':memory:')`. Run the full `ALL_DDLS` string from `src/main/store/db.ts` to set up schema, then call `runMigrations(db)` to add the confidence column. Instantiate `TranscriptStore(db)`.

Tests to write:

1. `createMeeting inserts a meetings row` — call `createMeeting('mtg-1', Date.now())`, then `db.prepare('SELECT id FROM meetings WHERE id = ?').get('mtg-1')` — expect row to exist.

2. `appendSegment writes a transcript_segments row with confidence` — create meeting first, then call `appendSegment` with a full `TranscriptSegmentRow` including `confidence: 0.92`. Query `db.prepare('SELECT confidence FROM transcript_segments WHERE id = ?').get(segmentId)` — expect `{ confidence: 0.92 }`.

3. `appendSegment with null confidence stores NULL` — same as above with `confidence: null`. Expect `{ confidence: null }`.

4. `getSegmentsByMeeting returns rows ordered by timestampStart` — insert 3 segments with timestamps 10.0, 5.0, 15.0 and verify returned array is ordered 5.0, 10.0, 15.0.

5. `runMigrations is idempotent` — call `runMigrations(db)` twice; no error thrown; `PRAGMA table_info` shows exactly one `confidence` column.

Import `Database` from `better-sqlite3-multiple-ciphers`. Note: `better-sqlite3-multiple-ciphers` is a native Node module rebuilt via `electron-rebuild`. In Vitest (Node environment, not Electron), it runs without SQLCipher encryption in `:memory:` mode — no `PRAGMA key` needed.
