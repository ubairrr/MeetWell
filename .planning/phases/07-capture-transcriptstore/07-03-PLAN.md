# Plan 07-03: audiotee system audio source + packaging

## Objective

Install `audiotee@0.0.7` and `@deepgram/sdk@5.4.0`, add the `extraResources` electron-builder directive so the `audiotee` binary is accessible outside the ASAR archive, and create `SystemAudioSource.ts` — the main-process class that wraps `AudioTee` and emits PCM buffers for `CaptureService` to forward to Deepgram. This plan runs in Wave 1 (parallel with 07-01 and 07-02) because it creates only new files and installs new packages.

## Context

**audiotee npm package API (verified from `node_modules/audiotee/dist/index.d.ts`):**

```typescript
interface AudioTeeOptions {
  sampleRate?: number        // Use 16000 — audiotee converts 48kHz natively to Int16LE
  chunkDurationMs?: number   // Use 250 — matches Deepgram streaming cadence
  binaryPath?: string        // REQUIRED in packaged app; see binary path resolution below
}

class AudioTee {
  on(event: 'data',  listener: (chunk: { data: Buffer }) => void): this
  on(event: 'start', listener: () => void): this
  on(event: 'stop',  listener: () => void): this
  on(event: 'error', listener: (error: Error) => void): this
  on(event: 'log',   listener: (level: string, msg: unknown) => void): this
  start(): Promise<void>
  stop(): Promise<void>
  isActive(): boolean
}
```

PCM format: `Int16LE` at `sampleRate: 16000`, mono. Each `data` event carries ~250ms = 8,000 bytes in `chunk.data`.

**Binary path resolution:** Phase 6 `electron-builder.yml` already has `resources/audiotee` in `asarUnpack`. An `extraResources` directive will copy `node_modules/audiotee/bin/audiotee` to `resources/audiotee` in the packaged app. In dev mode, the binary is at `node_modules/audiotee/bin/audiotee` relative to the project root.

**CAPT-03 fallback:** If `audiotee.start()` throws or emits an `error` event before the first `data` event, `SystemAudioSource` emits a `'fallback-needed'` event. `CaptureService` listens for this and activates the Chromium loopback path (a separate concern handled in 07-05 — `SystemAudioSource` just signals the failure). The Chromium fallback requires the renderer to call `getUserMedia` with system audio; `SystemAudioSource` does not implement the fallback itself.

**CAPT-09:** The `Buffer` received in the `data` event is forwarded immediately to the callback and then released. `SystemAudioSource` never stores or accumulates PCM beyond the current in-flight buffer.

**D-03 reconnect for audiotee:** If `audiotee` crashes after startup (emits `error` after `start` succeeded), `SystemAudioSource` attempts restart up to 3 times with a 2-second delay, matching the Deepgram reconnect policy (D-03). After 3 failures it emits `'error'` to notify `CaptureService` to update health status.

### Task 2: Add extraResources to electron-builder.yml

**File(s):** `electron-builder.yml`

**Action:** Modify

**Details:**

Add an `extraResources` section to `electron-builder.yml`. The Phase 6 config has `asarUnpack: ['resources/**', ...]` which includes `resources/audiotee`. The `extraResources` directive instructs electron-builder to copy the binary from `node_modules` into the resources directory before packaging:

```yaml
extraResources:
  - from: node_modules/audiotee/bin/audiotee
    to: audiotee
```

This places the binary at `{app}/Contents/Resources/audiotee` in the packaged app, which matches the `asarUnpack: 'resources/audiotee'` path. In production, it is accessed via `path.join(process.resourcesPath, 'audiotee')`.

Place `extraResources` at the top level of `electron-builder.yml` (same level as `asar`, `files`, etc.), after the `asarUnpack` block.

### Task 3: SystemAudioSource class

**File(s):** `src/main/capture/SystemAudioSource.ts`

**Action:** Create

**Details:**

```typescript
import { EventEmitter } from 'events'
import path from 'path'
import { app } from 'electron'
import { AudioTee } from 'audiotee'

export type SystemAudioSourceEvent = 'data' | 'start' | 'stop' | 'error' | 'fallback-needed'

export class SystemAudioSource extends EventEmitter {
  private audiotee: AudioTee | null = null
  private retryCount = 0
  private readonly MAX_RETRIES = 3
  private readonly RETRY_DELAY_MS = 2000
  private startedSuccessfully = false

  private getBinaryPath(): string {
    if (app.isPackaged) {
      return path.join(process.resourcesPath, 'audiotee')
    }
    // Dev: relative to this compiled file in out/main/
    return path.join(__dirname, '../../../node_modules/audiotee/bin/audiotee')
  }

  async start(): Promise<void> {
    this.startedSuccessfully = false
    this.retryCount = 0
    await this.startInternal()
  }

  private async startInternal(): Promise<void> {
    this.audiotee = new AudioTee({
      sampleRate: 16000,
      chunkDurationMs: 250,
      binaryPath: this.getBinaryPath(),
    })

    this.audiotee.on('start', () => {
      this.startedSuccessfully = true
      this.retryCount = 0
      this.emit('start')
    })

    this.audiotee.on('data', (chunk: { data: Buffer }) => {
      // CAPT-09: emit immediately, do not store the buffer
      this.emit('data', chunk.data)
    })

    this.audiotee.on('error', (err: Error) => {
      if (!this.startedSuccessfully) {
        // Failed before first start — signal fallback (CAPT-03)
        this.emit('fallback-needed', err)
        return
      }
      // Mid-session crash — attempt D-03 reconnect
      this.handleCrash(err)
    })

    this.audiotee.on('stop', () => {
      this.emit('stop')
    })

    this.audiotee.on('log', (_level: string, _msg: unknown) => {
      // Suppress debug logs in production
    })

    try {
      await this.audiotee.start()
    } catch (err) {
      if (!this.startedSuccessfully) {
        this.emit('fallback-needed', err)
      } else {
        this.handleCrash(err instanceof Error ? err : new Error(String(err)))
      }
    }
  }

  private async handleCrash(err: Error): Promise<void> {
    if (this.retryCount >= this.MAX_RETRIES) {
      this.emit('error', err)
      return
    }
    this.retryCount++
    await new Promise(r => setTimeout(r, this.RETRY_DELAY_MS))
    await this.startInternal()
  }

  async stop(): Promise<void> {
    if (this.audiotee?.isActive()) {
      await this.audiotee.stop()
    }
    this.audiotee = null
  }
}
```

**Type import note:** `AudioTee` from `audiotee` may not have full TypeScript declarations. If the import produces a type error, add `declare module 'audiotee'` shim or use `// @ts-expect-error` on the import line with a comment explaining the missing types.
