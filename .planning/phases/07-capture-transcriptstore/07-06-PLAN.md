# Plan 07-06: CapturingScreen UI — two-dot health bar + Stop Meeting

## Objective

Create the minimal `CapturingScreen` component (D-07, D-08): two colored dots representing mic and system audio health plus a "Stop Meeting" button. Wire it into `App.tsx` so it replaces `ConsentGateScreen` when `sessionState === 'Capturing'`. The `App.tsx` change also adds the `useCapturingHealth` hook that listens to `capture-health-update` IPC events and starts mic capture when the session enters `Capturing` state.

## Context

**D-07:** Two dots, one per channel. Colors map to `HealthStatus`:
- `'idle'` → gray (`#6b7280`)
- `'healthy'` → green (`#22c55e`)
- `'silent'` → yellow (`#eab308`)
- `'error'` → red (`#ef4444`)

**D-08:** `CapturingScreen` is shown when `sessionState === 'Capturing'`. It contains only the health bar and the "Stop Meeting" button. No transcript display, no live summary — those are Phase 9.

**Mic capture lifecycle:** The renderer must start mic capture (`startMicCapture()` from `MicCapture.ts`) when the session enters `Capturing` state and stop it when leaving. This is handled in `App.tsx` via a `useEffect` that watches `sessionState`. The `MicCapture.ts` module (07-04) returns a `MicCaptureHandle` with a `stop()` method — store it in a ref and call `stop()` on cleanup.

**IPC channels used (already in preload allowlist from Phase 6):**
- Listen: `capture-health-update` — payload `{ channel: 'mic'|'system', status: HealthStatus }`
- Invoke: `end-meeting` — fired by "Stop Meeting" button
- Listen: `session-state-changed` — already wired in `App.tsx`

**`window.electronAPI` type:** Phase 6 `src/preload/index.d.ts` or `env.d.ts` declares the type. The `invoke` and `on` signatures accept `string` at the preload level. Use `window.electronAPI.on('capture-health-update', handler)` — the handler receives the raw payload. Cast appropriately.

**This plan runs in Wave 1** because `CapturingScreen.tsx` and `ChannelHealthDot.tsx` are new files with no dependencies on 07-01 through 07-05. `App.tsx` is modified but only adds a new render branch (`sessionState === 'Capturing'`) — it does not conflict with files modified in other Wave 1 plans. The import of `MicCapture.ts` from 07-04 is a same-wave sibling; the executor must ensure 07-04 is committed before 07-06 modifies `App.tsx`, or accept a TypeScript path that resolves after 07-04 creates the file. Since both run in Wave 1 the executor can create both in sequence.

### Task 3: Update App.tsx — add useCapturingHealth hook and CapturingScreen branch

Replace the current `App.tsx` with an updated version that:

1. Adds `useCapturingHealth()` hook — listens to `capture-health-update` IPC events and maintains `healthMic` and `healthSystem` state:

```typescript
function useCapturingHealth(): { healthMic: HealthStatus; healthSystem: HealthStatus } {
  const [healthMic, setHealthMic] = useState<HealthStatus>('idle')
  const [healthSystem, setHealthSystem] = useState<HealthStatus>('idle')

  useEffect(() => {
    window.electronAPI.on('capture-health-update', (payload: unknown) => {
      const { channel, status } = payload as { channel: 'mic' | 'system'; status: HealthStatus }
      if (channel === 'mic') setHealthMic(status)
      else setHealthSystem(status)
    })
  }, [])

  return { healthMic, healthSystem }
}
```

2. Adds mic capture lifecycle management — a `useEffect` that starts `startMicCapture()` when `sessionState` becomes `'Capturing'` and stops it on cleanup or state change:

```typescript
const micHandleRef = useRef<MicCaptureHandle | null>(null)

useEffect(() => {
  if (sessionState !== 'Capturing') return

  let cancelled = false
  startMicCapture().then((handle) => {
    if (cancelled) {
      handle.stop()
    } else {
      micHandleRef.current = handle
    }
  }).catch((err: unknown) => {
    console.error('[App] mic capture failed:', err)
  })

  return () => {
    cancelled = true
    micHandleRef.current?.stop()
    micHandleRef.current = null
  }
}, [sessionState])
```

3. Renders `<CapturingScreen>` when `sessionState === 'Capturing'`:

```typescript
if (sessionState === 'Capturing') {
  return (
    <div id="overlay-root" style={{ width: '380px', minHeight: '100vh', background: 'rgba(0,0,0,0.85)', color: '#fff' }}>
      <CapturingScreen healthMic={healthMic} healthSystem={healthSystem} />
    </div>
  )
}
```
