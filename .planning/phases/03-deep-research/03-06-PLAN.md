# Plan: RSCH-04 — System-Audio Capture Spike

## Goal

Execute the hands-on throwaway capture spike comparing `electron-audio-loopback` (Chromium ScreenCaptureKit/Core Audio flags) and `AudioTee.js` (Core Audio Taps, Swift binary) on the current machine (macOS 26.5.1 Tahoe). The spike validates whether a coherent Deepgram Nova-3 transcript is returned for both mic and system audio channels simultaneously. The only deliverable that enters the planning repo is the SPIKE REPORT — the spike code is throwaway.

**This plan is `autonomous: false`** because the executor must:
1. Have a valid Deepgram API key to run the transcript round-trip (the spike's success signal)
2. Actually run code on the machine and observe results empirically

The user must provide `DEEPGRAM_API_KEY` before the executor starts the spike tasks (T3–T5).
